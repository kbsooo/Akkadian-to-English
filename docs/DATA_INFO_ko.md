# 데이터셋 안내

아카드어/고대 아시리아어 텍스트 작업에서 가장 큰 과제는 서식 문제를 다루는 것입니다. "쓰레기를 넣으면 쓰레기가 나온다(garbage in, garbage out)"는 말처럼, 음역 텍스트의 형식은 토큰화부터 변환 및 임베딩 과정까지 ML 워크플로우의 각 단계에서 어려움을 야기합니다.

이러한 문제를 완화하기 위해, 음역 텍스트와 번역 텍스트 모두에서 다양한 서식 문제를 처리하는 방법에 대한 정보와 제안을 아래에 제공합니다.

## 음역 텍스트

- 주요 서식 문제: 하이픈으로 연결된 음절의 표준 음역 형식 외에도, 아시리아학 전문가에게만 의미 있는 위 첨자, 아래 첨자, 구두점 등 추가적인 필사 표기가 텍스트를 복잡하게 만듭니다(전체 음역 변환 가이드 참조).
- 대문자 사용도 어려운 문제입니다. 두 가지 다른 방식으로 의미를 담고 있기 때문입니다. 단어의 첫 글자가 대문자이면 그 단어가 인명이나 지명(즉, 고유명사)임을 의미합니다. 단어가 모두 대문자이면 수메르어 표의문자(logogram)이며, 필사의 편의를 위해 아카드어 음절 표기 대신 사용된 것입니다.
- 한정사(Determinatives)는 아카드어에서 명사와 고유명사를 분류하는 일종의 분류자로 사용됩니다. 이 기호들은 보통 분류하는 명사 옆에 위 첨자 형식으로 인쇄됩니다. 한정사 기호를 단어의 일부로 읽는 혼란을 피하기 위해, 표준 음역 가이드를 따라 중괄호로 이를 유지했습니다. 이것이 ML에서 어려움을 줄 수 있지만, 음역에서 중괄호는 이 용도로만 사용된다는 점에 유의하세요(예: a-lim{ki}, A-mur-{d}UTU).
- 점토판의 손상된 텍스트: 고대 텍스트이므로 다수의 파손과 결손이 있습니다. 이러한 파손을 표준화하기 위해 두 가지 마커만 사용할 것을 제안합니다. 하나는 단일 기호의 작은 파손인 <gap>이고, 다른 하나는 하나 이상의 기호부터 큰 파손까지를 나타내는 <big_gap>입니다.
- 이 대회를 위해 아래에 이러한 서식 문제를 가장 잘 처리하는 방법에 대한 제안을 포함합니다.

## 번역 텍스트

현재 고대 설형문자 문서 번역에 대한 완전하거나 광범위한 데이터베이스는 없으며, 이는 특히 고대 아시리아어 텍스트에 해당합니다. 이러한 이유로 고대 아시리아어 텍스트의 번역과 주석이 담긴 책과 논문을 모아 OCR과 LLM을 사용하여 디지털화하고 수정했습니다. 그 모든 작업 후에도 이러한 번역에는 여전히 많은 서식 문제가 있어, 성공적인 기계 번역 개발을 위한 핵심 과제가 되고 있습니다.

번역에서는 일반적으로 동일한 고유명사 대문자 표기를 유지하며, 이러한 고유명사는 일반적으로 대부분의 ML 작업에서 성능이 떨어지는 부분입니다. 이러한 문제를 해결하기 위해, 전문가들이 인쇄 출판물을 위해 정규화한 모든 고유명사가 포함된 어휘집을 데이터셋에 포함했습니다.

### 현대 필사 표기법
마지막으로, 음역과 번역 텍스트에 동반되는 현대 필사 표기법이 있다는 점을 알아두는 것이 중요합니다. 첫 번째는 줄 번호입니다. 일반적으로 1, 5, 10, 15 등으로 번호가 매겨집니다. 그러나 파손된 줄이 있으면 줄 번호 바로 뒤에 아포스트로피(')가 붙고, 두 번째 파손된 줄 세트가 있으면 두 개의 아포스트로피('')가 붙습니다. 이것은 따옴표가 아니라 편집자들이 출판 시 가끔 사용하는 필사 관례입니다.

추가적인 필사 표기법:

- 느낌표는 학자가 어려운 기호 판독에 확신할 때 사용 !
- 물음표는 학자가 어려운 기호 판독에 불확실할 때 사용 ?
- 슬래시는 줄에 속한 기호가 줄 아래에 있을 때 사용 /
- 콜론은 고대 아시리아어 단어 구분 기호를 나타냄 :
- 파손과 삭제에 대한 주석은 괄호 안에 표시 ( )
- 필사 삽입 수정은 꺾쇠 괄호 안에 표시 < >
- 잘못되거나 오류 있는 기호는 이중 꺾쇠 괄호로 표시 << >>
- 부분적으로 파손된 기호는 반 괄호로 표시 ˹ ˺
- 명확히 파손된 기호와 줄은 대괄호로 표시 [ ]
- 한정사는 중괄호로 표시(아래 참조) { }

## 음역과 번역을 위한 서식 제안:

### 제거할 것 (현대 필사 표기법):
- ! (확실한 판독)
- ? (불확실한 판독)
- / (줄 구분자)
- : 또는 . (단어 구분자)
- < > (필사 삽입, 단 음역/번역의 텍스트는 유지)
- ˹ ˺ (부분 파손 기호, 음역에서 제거)
- [ ] (문서 수준 음역에서 제거. 예: [KÙ.BABBAR] → KÙ.BABBAR)

### 대체할 것 (파손, 공백, 위 첨자, 아래 첨자):
- [x] <gap>
- … <big_gap>
- [… …] <big_gap>
- ki {ki} (전체 목록은 아래 참조)
- il5 il5 (아래 첨자 숫자도 동일하게 적용)

### 중괄호로 표시된 아카드어 한정사:

# 데이터셋 설명

대회 데이터는 8,000개 이상의 고대 아시리아 설형문자 텍스트 음역과 종합적인 메타데이터 세트로 구성됩니다. 이 중 일부에 대해 정렬된 영어 번역을 제공합니다. 또한 추가 학습 데이터를 생성하는 데 사용할 수 있는 약 900개의 학술 출판물에서 가져온 미가공 텍스트도 제공합니다.

이러한 음역에 사용된 서식 규칙에 대한 자세한 내용은 데이터셋 안내를 참조하세요.

이것은 코드 대회입니다. test.csv의 데이터는 솔루션 작성을 돕기 위한 더미 데이터일 뿐입니다. 제출물이 채점될 때 이 예시 테스트 데이터는 전체 테스트 셋으로 대체됩니다.

## 파일 및 필드 정보
- `train.csv` - 원본 발굴 점토판에서 가져온 약 1,500개의 고대 아시리아어 텍스트 음역과 각각의 영어 번역.

    - `oare_id` - Old Assyrian Research Environment (OARE) 데이터베이스 식별자. 각 텍스트를 고유하게 식별합니다.
    - `transliteration` - 원본 점토판 텍스트의 아카드어 음역.
    - `translation` - 해당 영어 번역.
- `test.csv` - 테스트 데이터를 대표하는 작은 예시 세트. 제출물이 채점될 때 이 예시 테스트 데이터는 전체 테스트 셋으로 대체됩니다. 테스트 데이터에는 약 400개의 고유 문서에서 가져온 약 4,000개의 문장이 있습니다. 학습 데이터는 문서 수준에서 번역이 정렬되어 있지만, 테스트 데이터는 문장 수준에서 번역이 정렬되어 있습니다.

    - `id` - 각 문장의 고유 식별자.
    - `text_id` - 각 문서의 고유 식별자.
    - `line_start, line_end` - 원본 점토판 내 문장의 경계를 나타냅니다. 문서 내 문장 순서를 정합니다. 이 필드는 `1`, `1'`, `1''`와 같은 값을 가진 `str` 타입입니다. 데이터셋 안내의 현대 필사 표기법에서 줄 번호에 대한 설명을 참조하세요.
    - `transliteration` - 원본 점토판 텍스트의 아카드어 음역. 여러분의 목표는 해당 `translation`을 생성하는 것입니다.
- `sample_submission.csv` - 올바른 형식의 샘플 제출 파일. 자세한 내용은 평가 페이지를 참조하세요.

### 보충 데이터

- `published_texts.csv` - OARE 데이터베이스에 게시된 데이터베이스 및 박물관 기록의 메타데이터 필드와 카탈로그 정보가 포함된 약 8,000개의 고대 아시리아어 텍스트 음역. 이 식별자를 사용하여 연결된 웹사이트에서 추가 정보를 검색할 수 있습니다. 이 음역에는 번역이 제공되지 않습니다.

    - `oare_id` - `train.csv`와 동일한 OARE 데이터베이스 식별자.
    - `online transcript` - DPI 웹사이트에서 호스팅되는 음역 전사본 URL.
    - `cdli_id` - CDLI 웹사이트 식별자. 여러 ID는 막대 `|`로 구분됩니다.
    - `aliases` - 텍스트의 다른 출판된 레이블(예: 출판 번호, 박물관 ID 등). 여러 ID는 막대 `|`로 구분됩니다.
    - `label` - 텍스트의 주요 지정 레이블.
    - `publication_catalog` - 출판물과 박물관 기록에서 발견된 텍스트 레이블. 여러 ID는 막대 `|`로 구분됩니다.
    - `description` - 텍스트의 기본 설명.
    - `genre_label` - 텍스트에 할당된 기본 장르. 모든 텍스트에 제공되지는 않습니다.
    - `inventory_position` - 박물관에서 발견된 텍스트 레이블. 여러 ID는 막대 `|`로 구분됩니다.
    - `online_catalog` - CC-0 메타데이터와 이미지가 있는 예일 컬렉션 URL.
    - `note` - 전문가가 주석이나 번역을 위해 작성한 메모.
    - `interlinear_commentary` - 특정 줄에서 텍스트를 논의하는 출판물 참조.
    - `online_information` - 대영박물관의 텍스트 URL(이 이미지들은 대영박물관 저작권이며 CC가 아닙니다). 모든 텍스트에 제공되지는 않습니다.
    - `excavation_no` - 발굴에서 텍스트에 할당된 식별자.
    - `oatp_key` - Old Assyrian Text Project에서 할당한 식별자.
    - `eBL_id` - eBL 웹사이트 식별자.
    - `AICC_translation` - 최초로 온라인에 게시된 기계 번역 URL. 이 번역 대부분은 품질이 매우 낮습니다.
    - `transliteration_orig` - OARE 데이터베이스의 원본 텍스트 음역.
    - `transliteration` - 서식 제안에 기반한 정제된 텍스트 음역 버전.
- `publications.csv` - 고대 아시리아어에서 여러 현대 언어로의 번역이 포함된 약 880개의 학술 출판물 원문 텍스트. OCR과 LLM 후처리를 통해 생성되었습니다. 이 번역을 추출하여 `published_texts.csv`의 음역과 정렬하는 시도를 할 수 있습니다. 번역이 영어가 아닌 다른 언어로 제공되는 경우가 많습니다.
    - `pdf_name` - 텍스트가 추출된 PDF 파일 이름.
    - `page` - 해당 텍스트가 나타난 페이지 번호.
    - `page_text` - 논문 자체의 텍스트.
    - `has_akkadian` - 텍스트에 아카드어 음역이 포함되어 있는지 여부.
- `bibliography.csv` - `publications.csv`의 텍스트에 대한 서지 데이터.
    - `pdf_name` - `publications.csv`의 ID에 해당.
    - `title`, `author`, `author_place`, `journal`, `volume`, `year`, `pages` - 표준 서지 데이터.
- `OA_Lexicon_eBL.csv` - 이 파일에는 모든 고대 아시리아어 단어의 음역과 어휘 등가어(사전에서 찾을 수 있는 형태) 목록이 포함되어 있습니다. 포함된 링크는 LMU에서 호스팅하는 온라인 아카드어 사전인 electronic Babylonian Library (eBL)로 연결됩니다.
    - `type` - 단어 유형(예: word, `PN` = 인명, `GN` = 지명).
    - `form` - 음역에서 발견되는 문자 그대로의 단어.
    - `norm` - 하이픈이 제거되고 모음 길이가 표시된 정규화 형태.
    - `lexeme` - 사전에서 찾을 수 있는 기본형.
    - `eBL` - electronic Babylonian Library (eBL) 온라인 사전 URL.
    - `I_IV` - eBL에서 찾을 수 있는 Concise Dictionary of Akkadian (CDA)에 해당하는 동음이의어 기본형의 로마 숫자 지정.
    - `A_D` - Chicago Assyrian Dictionary에 해당하는 동음이의어 기본형의 알파벳 지정.
    - `Female(f)` - 여성 성별 지정.
    - `Alt_lex` - 대체 정규화.
- `eBL_Dictionary.csv` - eBL 데이터베이스의 완전한 아카드어 단어 사전. `OA_Lexicon_eBL.csv`의 `eBL` URL에서 제공하는 데이터를 수집한 것입니다.
- `resources.csv` - 추가 데이터로 사용할 수 있는 리소스 목록.
- `Sentences_Oare_FirstWord_LinNum.csv` - `train.csv` 데이터의 문장 수준 번역 정렬을 돕는 파일. 각 문장의 첫 단어와 점토판 내 위치를 나타냅니다.

### 추가 학습 데이터 구축을 위한 권장 워크플로우
`publications.csv` 파일에는 약 900개 PDF의 OCR 출력이 포함되어 있으며, 이 텍스트에서 번역을 추출하는 것이 필수적인 첫 단계입니다. 머신러닝을 시작하기 전에 학습 데이터를 재구성하고 정렬해야 합니다. 다음은 따를 수 있는 간단한 경로입니다:

1. 각 텍스트와 번역 찾기: 문서 식별자(ID, 별칭 또는 박물관 번호)를 사용하여 OCR 출력에서 음역을 해당 번역과 매칭합니다.

2. 모든 번역을 영어로 변환: 원본 번역은 여러 언어(예: 영어, 프랑스어, 독일어, 터키어)로 나타날 수 있습니다. 일관성을 위해 모든 것을 영어로 변환합니다.

3. 문장 수준 정렬 생성: 아카드어 음역과 매칭되는 영어 번역을 모두 문장 단위로 분할하고 쌍으로 정렬합니다. 이 문장 수준 매핑은 기계 번역 모델을 학습하고 평가하는 데 가장 유용한 형식입니다.

이러한 단계가 완료되면 머신러닝에 사용할 준비가 된 깔끔하고 정렬된 데이터셋을 갖게 됩니다.

## 참고 문헌
참고 문헌은 대회용 번역을 검색하는 데 사용한 2차 자료를 반영합니다. 각 자료는 기계 번역 생성 시 사용되었다면 인용되어야 합니다.
